# Contributing

## Branching
- Use `feat/*` for new functionality.
- Use `fix/*` for bug fixes.
- Use `chore/*` for maintenance or infrastructure changes.

## Commits
- Follow Conventional Commits (`type(scope): summary`).
- Keep messages imperative and under 72 characters when possible.

## Pull Requests
- Ensure one reviewer approval and passing CI checks.
- Keep scope focused; reference relevant issues or ADRs.
- Update documentation when workflow or governance changes.
